# 🌿 Branch-Based Repository Structure

Instead of storing everything in the **main** branch, this repository is organized using **separate Git branches**.

Each branch represents a different learning module or technology.

This keeps the repository clean, organized, and easy to navigate.

### Available Branches

| Branch | Description |
|---------|-------------|
| `Core_JAVA` | Core Java concepts and programming exercises |
| `Data_Structures_And_Algorithms` | DSA practice and coding problems |
| `Design_Patterns_And_Principles` | Design Patterns and SOLID Principles |
| `main` | Repository overview and documentation |

> Additional branches will be created as new modules are covered during the training.

---

# 📂 Why Separate Branches?

Using dedicated branches provides several advantages:

- Better organization of module-wise code
- Independent version history for each topic
- Easy navigation between learning modules
- Cleaner repository structure
- Simplified maintenance and updates
